# 2.2 - The Audio System

### Audio Pool

The Audio System works using the audio pool scriptable object which can be assigned in the inspector. The audio pool contains entries which decide key info about sound effects: their name, clip, volume, pitch etc.

---

#### Audio Source Setups

Normally, when an Audio Source is available, you can play audio with the `Play` function. All `Play` functions use the `SourceSetup` function to set the values of the source based upon the selected entry.

- Looping is set
- Source volume is set
- The source's spacial blend is set to one if a position is given and the entry's spatial blend is zero, otherwise it sets to the entry's blend.
- The source's transform position is updated if a position is given
- If no position is given the source's spatial blend is set to zero.

```
private static void SourceSetup(AudioSource source, AudioPoolEntry entry, Vector3? pos, bool loop = false)
{
    source.loop = loop;
    source.volume = entry.localVolume;

    if (pos.HasValue)
    {
        source.spatialBlend = entry.spatialBlend != 0.0f ? entry.spatialBlend : 1.0f;
        source.transform.position = pos.Value;
    }
    else
    {
        source.spatialBlend = 0.0f;
    }

    return;
}
```

#### Playing Audio

- `Play()` plays given the source.
- `PlayOneShot()` plays given the source, without looping or pitch changing.
- `PlayCreateInstance()` creates an audio source game object and plays a sound from it, with a choice to destroy it when completed.
