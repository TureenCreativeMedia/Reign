# 1.3 - Design

Reign's design follows a simple philosophy that it should always be persistent throughout the game's runtime using its prefab.

Reign is made up of [systems](1.2.md), each requiring their own requisites.

---

### Reign Prefab
The Reign prefab is a **Don't Destroy On Load Singleton** following the inclusion of `Reign.cs`, it should remain present between every scene. The prefab contains a child, 'Systems', which has all of the systems as scripts attached to it.

---

### Adding Reign To New Scenes

Every scene should always contain the Reign scene dependencies. To combat confusion, Reign Tools (Reign/Reign Tools from the toolbar) allows for the user to add everything required to the current scene.

---

### Reign vs. Your Project

On modern versions, Reign is stored in `/Assets/Plugins`, where as your project is stored in just `/Assets`.
